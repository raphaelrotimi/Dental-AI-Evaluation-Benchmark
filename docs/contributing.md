# Contributing

## Dental AI Evaluation Benchmark (DAEB)

Thank you for your interest in contributing to the Dental AI Evaluation Benchmark (DAEB).

This project aims to provide a structured, reproducible benchmark for evaluating Large Language Model (LLM) responses to dental healthcare prompts using standardized human evaluation.

---

# Project Goals

Contributions should support one or more of the following goals:

- Improve benchmark quality.
- Expand dental topic coverage.
- Enhance documentation.
- Improve evaluation consistency.
- Maintain clinical safety.
- Promote evidence-based educational content.

---

# Contribution Guidelines

Before contributing, please ensure that all proposed changes:

- Follow the existing repository structure.
- Maintain the standardized evaluation template.
- Preserve consistent formatting.
- Use professional and objective language.
- Prioritize clinical safety.

---

# Dataset Entry Requirements

Each dataset entry should include:

1. Evaluation Metadata
2. Clinical Scenario
3. User Prompt
4. Three Candidate Responses
5. Response Ranking
6. Rubric Evaluation
7. Final Evaluation Summary
8. Overall Winner
9. Evaluation Confidence
10. Key Evaluation Takeaways
11. Skills Demonstrated
12. Evaluation Tags

All entries should follow this structure to ensure consistency across the benchmark.

---

# Evaluation Standards

Contributors should evaluate responses using the established rubric:

- Accuracy
- Instruction Following
- Completeness
- Clarity
- Safety

Responses should be ranked according to their overall quality, not their length.

---

# Clinical Safety

Healthcare-related benchmark entries should:

- Avoid unsupported diagnoses.
- Avoid inappropriate treatment recommendations.
- Encourage professional evaluation when appropriate.
- Include emergency warning signs when relevant.
- Provide educational information rather than individualized medical advice.

---

# Documentation Standards

Documentation should be:

- Clear
- Consistent
- Well organized
- Grammatically correct
- Written in professional English

Markdown formatting should remain consistent throughout the repository.

---

# Commit Message Guidelines

Use concise, descriptive commit messages.

Examples:

```text
Add Dataset Entry 031: Dental implant maintenance evaluation benchmark

Update README with benchmark statistics

Add scoring rubric documentation for Benchmark Release v1.0
```

---

# Pull Requests

When submitting a pull request:

- Clearly describe the proposed changes.
- Explain the motivation for the update.
- Ensure documentation remains consistent.
- Verify that formatting has been preserved.

---

# Code of Conduct

Contributors are expected to:

- Be respectful.
- Provide constructive feedback.
- Support evidence-based healthcare information.
- Maintain a professional tone in all discussions.

---

# Questions

If you have questions about the benchmark design or evaluation methodology, please open a GitHub Issue for discussion.

---

# Thank You

Thank you for helping improve the Dental AI Evaluation Benchmark and supporting responsible AI evaluation in dental healthcare.
